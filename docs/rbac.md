# rbac роли и права доступа

в этом репозитории backend реализован на spring boot java а frontend на react javascript
в задании упомянут fastapi и react typescript но принципы rbac и проверка прав реализованы эквивалентно на текущем стеке

## сущности и сценарии

сущности
- user профиль пользователя
- book книга в каталоге
- exchange_request заявка на обмен
- user_ratings оценка участника обмена
- refresh_token refresh token

ключевые действия
- просмотр каталога и карточки книги
- регистрация и вход
- создание книги
- редактирование и удаление книги
- просмотр профилей пользователей
- редактирование своего профиля и смена пароля
- создание заявки на обмен
- просмотр входящих и исходящих заявок
- принять отклонить отменить заявку
- поставить оценку по завершенному обмену
- администрирование ролей

## роли

минимум ролей
- guest неавторизованный пользователь
- user авторизованный пользователь
- admin администратор

## permissions и ограничения

ограничения по данным
- user может редактировать и удалять только свой профиль
- user может редактировать и удалять только свои книги
- user видит только свои входящие и исходящие заявки на обмен
- user может принять или отклонить только заявку где он получатель
- user может отменить только заявку где он инициатор
- user может поставить оценку только если он участник обмена и еще не оценивал этот обмен

## реализация на backend

хранение роли
- добавлено поле role в таблицу users и в модель пользователя
- значения роли USER или ADMIN

проверка прав
- запрет по умолчанию все запросы кроме белого списка требуют аутентификацию
- public маршруты
  - /api/auth
  - post /api/users регистрация
  - get /api/users/{id} публичный профиль
  - get /api/books и /api/books/{id} каталог
- admin маршруты
  - get /api/users список пользователей
  - /api/admin

возврат ошибки
- при недостатке прав возвращается 403

endpoint для управления ролями
- put /api/admin/users/{id}/role body { "role": "ADMIN" }

дополнительный endpoint для frontend
- get /api/users/me возвращает текущего пользователя включая role

## реализация на frontend

- приватные маршруты закрыты компонентом RequireAuth
- роль пользователя берется через /api/users/me после login или refresh
- добавлены компоненты RequireAuth и RequireRole как базовая инфраструктура для ролевых роутов

## как проверить

1 войти под первым зарегистрированным пользователем он станет admin
2 запросить список пользователей get /api/users должен работать только для admin
3 попытаться вызвать put /api/admin/users/{id}/role под обычным user должен быть 403
4 открыть /add-book без логина должно редиректить на /login
