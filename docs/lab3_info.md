# Лабораторная работа №3: Фильтрация, Пагинация и Хранилище S3

В рамках данной лабораторной работы была реализована система продвинутого поиска книг, постраничный вывод данных и интеграция с объектным хранилищем для работы с изображениями.

## Что сделано

### 1. Бэкенд (Spring Boot)
- **Фильтрация и Поиск**: В `BookRepository` реализован динамический SQL-запрос, поддерживающий фильтрацию по названию, автору, жанру, состоянию и статусу книги.
- **Пагинация**: Добавлена поддержка параметров `page` и `size` на уровне контроллера и репозитория (`LIMIT` и `OFFSET`).
- **Объектное хранилище (S3)**: 
    - Создан `S3Service` с использованием AWS SDK v2 для работы с S3-совместимыми хранилищами (MinIO, LocalStack).
    - Обложки книг теперь загружаются как файлы через `multipart/form-data` и сохраняются в бакет `book-covers`.
- **Валидация**: Сущность `Book` дополнена Bean-валидацией (`@NotBlank`, `@Size`), которая проверяется при создании и обновлении.

### 2. Фронтенд (React)
- **Интерфейс фильтрации**: В каталоге добавлена панель фильтров с поиском и выпадающими списками.
- **Состояние в URL**: Все примененные фильтры и номер страницы сохраняются в `query-параметрах` браузера, что позволяет делиться ссылкой на конкретную выборку.
- **Пагинация**: Реализован блок управления страницами.
- **Загрузка файлов**: Форма добавления/редактирования книги теперь поддерживает выбор файла с локального устройства, который затем отправляется на S3 через бэкенд. Добавлен предпросмотр выбранного изображения.

## Как проверить

### Фильтрация и Пагинация
1. Откройте страницу **Каталог**.
2. Введите текст в строку поиска или выберите жанр/состояние.
3. Убедитесь, что запрос на бэкенд содержит параметры (вкладка *Network* -> `api/books?query=...&genre=...`).
4. Попробуйте переключить страницы внизу списка.

### Загрузка обложки в S3
1. Перейдите на страницу **Добавить книгу**.
2. Заполните данные и выберите файл в поле "обложка книги".
3. После сохранения убедитесь, что в каталоге отображается именно ваша картинка.
4. В БД `cover_url` теперь будет иметь вид `http://localhost:4566/book-covers/...`.

## Конфигурация
Для работы S3 (LocalStack) в `application.properties` используются следующие настройки:
- `s3.endpoint`: URL хранилища (дефолт для LocalStack: http://localhost:4566)
- `s3.bucket`: название корзины (`book-covers`)
- `spring.servlet.multipart`: лимит на размер файла (10MB)
