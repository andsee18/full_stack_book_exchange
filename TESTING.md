# Тестирование проекта BookExchange

В рамках лабораторной работы №5 реализовано комплексное тестирование клиентской и серверной частей приложения.

## 1. Backend Тестирование (Spring Boot)

### Модульные тесты (Unit Tests)
- **UserServiceTest**: Тестирование бизнес-логики пользователей (регистрация, аутентификация).
- Используется **JUnit 5** и **Mockito**.

### Интеграционные тесты (Integration Tests)
- **AuthControllerIntegrationTest**: Проверка API эндпоинтов авторизации.
- Используется **MockMvc** для имитации HTTP-запросов и **@SpringBootTest**.

### Запуск тестов бэкенда:
```bash
cd bookexchange_backend
.\mvnw.cmd test
```

## 2. Frontend Тестирование (React)

### Модульные тесты утилит
- **jwt.test.js**: Проверка корректности декодирования JWT токенов.

### Тесты компонентов и страниц
- **Login.test.js**: Проверка рендеринга формы входа и обработки ошибок.
- Используется **React Testing Library** и **Jest**.

### Мокирование API
- Настроена библиотека **MSW (Mock Service Worker)** для имитации ответов сервера в тестах.

### Запуск тестов фронтенда:
```bash
cd frontend
npm test
```

## 3. Сценарии тестирования (Тестовая модель)
1. **Регистрация**: Успешное создание, ошибка при дубликате username.
2. **Авторизация**: Вход с верными/неверными данными, защита маршрутов.
3. **Обмен книг**: (E2E) Подача заявки, изменение статуса книги.
4. **Внешнее API**: Обработка ответов от Open Library API.

## 4. Метрики
- Покрытие тестами критических узлов (Auth, User Service).
- Использование изолированного окружения (MockMvc, MSW).
